## ツイート 2（コンセプト）

KAIJU VOICE のコンセプト:

「声の大きさ」だけじゃない。
Geminiが分析するのは3つの軸:

- intensity（迫力）: 声量・勢い
- creativity（創造性）: 技名のユニーク度・言葉遊び
- emotion（感情）: 叫びに込められた感情

日英混在で叫ぶと「創造性ボーナス」が入る仕様。
会場では「ファイヤーーー！Burn everything!!」が飛び交ってました。

## ツイート 3（Gemini 3モデル活用）

1つのゲームに Gemini API を3つ組み合わせました:

1. gemini-3-flash-preview — 音声分析（叫び声→3軸スコアリング）
2. gemini-3-flash-preview — AI対戦ロジック（HP比率で感情が変わる攻撃生成、temperature: 1.0）
3. imagen-4.0-generate-001 — 怪獣・ヒーローの画像を動的生成

各APIを独立した「専門エージェント」として設計し、プリフェッチで応答を先読みすることでリアルタイム体験を実現。

## ツイート 4（AI対戦相手の設計）

AI対戦相手が面白い。
Geminiに「感情を持つ対戦相手」を演じさせています:

- HP > 50%: 互角の戦い。力強く攻撃
- HP < 50%: 追い詰められて焦りと怒り
- HP < 25%: 絶体絶命。必死で必殺技「絶零砲ォォォ！！」
- 相手HP < 25%: 圧倒的優勢。余裕と高揚で畳み掛ける

属性ごとに技名ヒント（火→灼熱・業火・爆炎、氷→氷結・絶零・凍牙...）も渡してる。
これがStructured Output + temperature 1.0で毎回違う叫びが返ってくる。

## ツイート 5（Geminiマルチモデル設計の裏側）

1つのゲームで3つのGemini APIを「専門エージェント」として使い分けた。

- 音声分析エージェント: gemini-3-flash + Structured Output → 確実にパース可能なJSON
- 対戦AIエージェント: gemini-3-flash + temperature 1.0 → 毎回違う叫びを生成
- 画像生成エージェント: Imagen 4 → 属性別の怪獣ビジュアル

各APIを独立したエージェントとして設計し、プリフェッチで応答を先読みすることでリアルタイム体験を実現。

## ツイート 6（技術的工夫）

地味だけど効いた技術的工夫:

「AIプリフェッチ」

プレイヤーが録音している5秒間に、裏でAIの次の攻撃を先読み生成しておく。

useRefにキャッシュ → consumePrefetched()で取り出し。

プリフェッチ済み: 300msで攻撃表示
ライブ生成フォールバック: 800ms

この差が体感テンポを劇的に改善。
